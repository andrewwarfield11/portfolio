This is a game that I made in java. The player controls the cannon on top of the tower and is try to defend against the invaders. Click on the screen and the cannon will shoot in that direction. The helicopters will constantly shoot at the tower, damaging it, and the enemies on the ground will walk towards the tower and attack it when they reach it.

There are ten levels total and in between each level the player can buy upgrades in a shop. Press on the name of an item to buy it.

Code Snippets

This is a screenshot from the bubble sort implementation. The tutorial would highlight two consecutive values in the list, with the variable tracker keeping track of the current position in the list of. If the user presses the swap button, swap will be called, swapping the two values highlighted and updating the tracker. If the user presses the pass button, it will just update the tracker. Every frame, there is a check to determine of the tracker is a different value than the first value highlighted, and if so the values highlighted will also be updated.

This is a screenshot showing the core of the stack implementation: pushing, popping, and peeking. Pushing will make sure that the top value is not being peeked at, get rid of any error messages that may have been created from trying pop or peek from an empty stack, then actually push the new node to the stack. Pop will show an error if the stack is empty, otherwise destroy the head value. Peek will also show an error if the stack is empty, and otherwise it will show the value of the head, or stop showing the value if peek had already been pressed.

Code Snippets

This snippet shows what happens at the beginning of this implementation of aes. The init function will len, which is the size of the key, to set the key length and number of rounds. Len is measured in bytes, so if it is 16 that means you are using 128 bit encryption, which means there will be 10 rounds. If len is 24, you are using 192 bit encryption and need 12 rounds. If len 32, you are using 256 bit encryption and need 14 rounds. The key length in this implementation is measured in units of 32 bit words, or 4 byte words. This means that the value used will always be len/4. Key expansion will then be called, which will then expand the key so that you have one 32 bit word for each of the rounds.
